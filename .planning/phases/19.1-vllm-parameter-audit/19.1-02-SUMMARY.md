## Decisions Made

- Dict-then-instantiate pattern for `_build_sampling_params()` — a single `return sampling_params_cls(**kwargs)` at the end makes the VLLMSamplingConfig override injection clean (no duplicate paths for greedy vs sampling)
- `get_backend_capabilities()` reads `VLLMEngineConfig.model_fields` — the capability matrix checks for field *existence* to determine support; the container `VLLMConfig` only has `engine` and `sampling`, not the actual capability fields
- `speculative_config` dict format — vLLM v0.6+ removed the direct `speculative_model` kwarg and requires the dict API; wrapping both fields into the dict at construction time is correct
