## Decisions Made

- Nested VLLMConfig mirrors vLLM's own API separation (LLM constructor vs SamplingParams) — makes it natural for researchers familiar with the vLLM API
- `gpu_memory_utilization` uses `lt=1.0` (strict less-than) rather than `le=1.0` — value of exactly 1.0 would leave no headroom for model weights
- Local `_unflatten`/`_deep_merge` helpers added to `grid.py` rather than importing from `config.loader` — avoids circular import (loader imports grid, grid cannot import loader)
