- `_BUILTIN_DIR = Path(__file__).parent / "builtin"` (safe for hatch src-layout editable install)
- `BUILTIN_DATASETS: dict[str, Path]` mapping `"aienergyscore"` to the JSONL path
- `AUTO_DETECT_COLUMNS = ["prompt", "text", "instruction", "input", "question"]`
- `load_prompts(config: ExperimentConfig) -> list[str]` function that:
  - If `config.dataset` is a `SyntheticDatasetConfig`, delegates to `_load_synthetic(config.dataset, config.n)`
  - If `config.dataset` is a string in `BUILTIN_DATASETS`, calls `_load_jsonl(path, config.n, name)`
  - If `config.dataset` is a string path to an existing `.jsonl` file, loads it
  - Otherwise raises `ValueError` with helpful message listing valid built-ins
- `_load_jsonl(path, n, name)` that:
  - Opens the file, skips lines starting with `{"_provenance"` (provenance header)
  - Reads prompt field from each JSON line (tries `"prompt"` then `"text"`)
  - Supports `dataset_order` from config if present (interleaved=file order, grouped=sorted by source, shuffled=seed-based)
  - Returns exactly `n` prompts, raises `ValueError` if fewer available
- `_load_synthetic(config, n)` that generates deterministic synthetic prompts from seed (same logic as current M1 placeholder but using `random.Random(config.seed)`)
- Use lazy import for `SyntheticDatasetConfig` inside `load_prompts` to avoid circular imports
