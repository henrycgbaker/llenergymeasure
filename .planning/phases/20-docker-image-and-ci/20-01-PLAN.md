**docker/Dockerfile.vllm — complete replacement:**

```dockerfile
# syntax=docker/dockerfile:1
# ============================================================
# vLLM backend — llenergymeasure on top of official vLLM image
# ============================================================
# Extends vllm/vllm-openai with llenergymeasure installed.
# CUDA and vLLM compatibility is guaranteed by the upstream image.
#
# Usage:
#   docker build -f docker/Dockerfile.vllm -t llenergymeasure-vllm .
#
# Version pin policy: update VLLM_VERSION on each milestone release.
# Must be compatible with pyproject.toml vllm>=0.6 constraint.
# ============================================================
ARG VLLM_VERSION=v0.7.3

FROM vllm/vllm-openai:${VLLM_VERSION}

# Reset upstream ENTRYPOINT — vLLM sets ENTRYPOINT to its API server,
# which conflicts with DockerRunner's command override pattern.
ENTRYPOINT []

WORKDIR /app

# Install llenergymeasure from source (no PyPI yet).
# --no-deps on the package itself avoids re-installing vLLM/torch.
# Then install remaining runtime deps that aren't in the vLLM image.
COPY pyproject.toml README.md ./
COPY src/ ./src/
RUN pip install --no-cache-dir --no-deps ".[vllm]" \
    && pip install --no-cache-dir \
        pydantic loguru typer pyyaml platformdirs nvidia-ml-py numpy \
        pyarrow tqdm codecarbon datasets python-dotenv

# HuggingFace cache — writable for non-root users
ENV HF_HOME=/app/.cache/huggingface
RUN mkdir -p /app/.cache/huggingface && chmod 777 /app/.cache/huggingface

CMD ["python", "-m", "llenergymeasure.infra.container_entrypoint"]
```

Key design decisions reflected:
- `FROM vllm/vllm-openai:${VLLM_VERSION}` — upstream-first, no shared base
- `ENTRYPOINT []` — critical for DockerRunner compatibility (the command after the image name in `docker run` overrides CMD, not ENTRYPOINT)
- `--no-deps` on `.[vllm]` — vLLM and torch are already in the base image
- Remaining deps listed explicitly — only what llenergymeasure needs at runtime that isn't already in the vLLM image
- `CMD` set to container_entrypoint — serves as default but DockerRunner overrides it explicitly

**docker/Dockerfile.pytorch — complete replacement:**

```dockerfile
# syntax=docker/dockerfile:1
# ============================================================
# PyTorch backend — llenergymeasure on top of official PyTorch image
# ============================================================
# Extends pytorch/pytorch with llenergymeasure installed.
# CUDA and PyTorch compatibility is guaranteed by the upstream image.
#
# Usage:
#   docker build -f docker/Dockerfile.pytorch -t llenergymeasure-pytorch .
#
# Version pin policy: update PYTORCH_VERSION on each milestone release.
# Must be compatible with pyproject.toml torch>=2.5 constraint.
# ============================================================
ARG PYTORCH_VERSION=2.5.1-cuda12.4-cudnn9-runtime

FROM pytorch/pytorch:${PYTORCH_VERSION}

WORKDIR /app

# Install llenergymeasure with pytorch extras from source (no PyPI yet).
# sentencepiece needed for Llama/Qwen tokenisers.
COPY pyproject.toml README.md ./
COPY src/ ./src/
RUN pip install --no-cache-dir ".[pytorch]" sentencepiece

# HuggingFace cache — writable for non-root users
ENV HF_HOME=/app/.cache/huggingface
RUN mkdir -p /app/.cache/huggingface && chmod 777 /app/.cache/huggingface

CMD ["python", "-m", "llenergymeasure.infra.container_entrypoint"]
```

Key design decisions reflected:
- `FROM pytorch/pytorch:${PYTORCH_VERSION}` — upstream-first, `-runtime` variant (no nvcc needed for inference)
- No ENTRYPOINT reset needed — PyTorch official image does not set a custom ENTRYPOINT
- Full `.[pytorch]` install with deps — torch comes from the base image, pip resolves against it
- `sentencepiece` added explicitly — needed for Llama/Qwen tokenisers, not in pyproject.toml pytorch extras

- Add PUID/PGID entrypoint script (DockerRunner doesn't use it)
