```python
def detect_default_backend() -> str:
    """Detect the default available backend. Returns 'pytorch' if transformers installed."""
    if importlib.util.find_spec("transformers") is not None:
        return "pytorch"
    raise BackendError("No inference backend installed...")

def get_backend(name: str) -> InferenceBackend:
    if name == "pytorch":
        from llenergymeasure.core.backends.pytorch import PyTorchBackend
        return PyTorchBackend()
    raise BackendError(f"Unknown backend: {name!r}. Available: pytorch")
```

  <name>Task 1: Write VLLMBackend class with offline batch inference lifecycle</name>
  <files>src/llenergymeasure/core/backends/vllm.py</files>
  <action>
Create `src/llenergymeasure/core/backends/vllm.py` — a ground-up implementation (do NOT copy from `core/inference_backends/vllm.py`). The class must satisfy the `InferenceBackend` protocol (name property + run method).

**Structure mirrors PyTorchBackend lifecycle but uses vLLM's offline batch API:**

1. **`name` property** → returns `"vllm"`

2. **`run(self, config: ExperimentConfig) -> ExperimentResult`** — the full lifecycle:
   - (1) `collect_environment_snapshot()` — lazy import from `domain.environment`
   - (2) `measure_baseline_power()` if `config.baseline.enabled` — lazy import from `core.baseline`
   - (3) `_load_model(config)` → returns `(llm, sampling_params)` where `llm` is `vllm.LLM` instance
   - (4) `_prepare_prompts(config)` → M1 placeholder (same pattern as PyTorchBackend: `"Hello, " * words_per_prompt` repeated `config.n` times)
   - (5) Warmup: `llm.generate(prompts[:1], SamplingParams(max_tokens=1, temperature=0.0))` — single prompt, 1 token, minimal warmup
   - (6) Thermal floor: `time.sleep(config.warmup.thermal_floor_seconds)` if enabled
   - (7) `select_energy_backend(config.energy.backend)` + `start_tracking()`
   - (8) `torch.cuda.synchronize()` before measurement start
   - (9) `llm.generate(prompts, sampling_params)` — single call with ALL prompts. Measure `time.perf_counter()` around this. Record `datetime.now()` for start/end times.
   - (10) `torch.cuda.synchronize()` after inference, before stopping energy
   - (11) `energy_backend.stop_tracking(tracker)`
   - (12) Count tokens: `sum(len(o.prompt_token_ids) for o in outputs)` for input, `sum(len(o.outputs[0].token_ids) for o in outputs)` for output
   - (13) FLOPs estimation — vLLM doesn't expose a HuggingFace model object with `.config`, so wrap `estimate_flops_palm` in a try/except and default to `None` on failure. Log a debug message if FLOPs estimation is skipped.
   - (14) `PowerThermalSampler` — wrap the generate call with `PowerThermalSampler(device_index=0)` context manager (same as PyTorchBackend) for timeseries and thermal info
   - (15) Write timeseries Parquet if `config.output_dir` is set
   - (16) `_collect_warnings()` — same pattern as PyTorchBackend
   - (17) `_build_result()` → construct `ExperimentResult` with `backend="vllm"`
   - **finally block**: `del llm` + `gc.collect()` + `torch.cuda.empty_cache()` (best-effort)

3. **`_load_model(config)` method:**
   - Build kwargs via `_build_llm_kwargs(config)`
   - `from vllm import LLM, SamplingParams` — lazy import
   - `llm = LLM(**kwargs)`
   - Build `SamplingParams` from `config.decoder` (temperature, top_p, top_k, max_tokens=config.max_output_tokens). For greedy: if `temperature == 0.0` or `do_sample is False`, set `temperature=0.0`. Convert `top_k=0` (our disabled sentinel) to vLLM's `-1` (disabled sentinel).
   - Return `(llm, sampling_params)`

4. **`_build_llm_kwargs(config)` method:**
   - Start with `{"model": config.model, "dtype": _map_precision(config.precision), "trust_remote_code": True, "seed": config.random_seed}`
   - Apply `VLLMConfig` fields if `config.vllm is not None`: `tensor_parallel_size`, `gpu_memory_utilization`, `max_num_seqs`, `enable_prefix_caching`, `quantization` — only set if not None
   - Return the kwargs dict

5. **`_map_precision(precision)` static method:**
   - `{"fp32": "float32", "fp16": "float16", "bf16": "bfloat16"}.get(precision, "auto")`

6. **`_build_result()` method** — same signature pattern as PyTorchBackend's `_build_result()`. Key differences: `backend="vllm"`, `total_flops` may be 0.0 if FLOPs estimation fails.

7. **`_cuda_sync()` static method** — identical to PyTorchBackend (best-effort `torch.cuda.synchronize()`).

8. **`_cleanup(llm)` method** — `del llm`, `gc.collect()`, best-effort `torch.cuda.empty_cache()`.

9. **`_collect_warnings()` method** — reuse `collect_measurement_warnings()` from `core.measurement_warnings` (same as PyTorchBackend).

**Important constraints:**
- All vLLM imports (`vllm.LLM`, `vllm.SamplingParams`) must be lazy (inside method bodies), not at module level. This allows the module to be imported on hosts without vLLM installed.
- All `torch` imports must be lazy for the same reason.
- Use `from __future__ import annotations` for type hints.
- No streaming implementation — this resolves CM-07 structurally.
- Use `logging.getLogger(__name__)` for all log messages.
- Follow the exact same docstring style as `pytorch.py`.
