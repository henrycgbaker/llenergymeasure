    **Tone:** Accessible, explanatory. No assumed ML or programming knowledge. Use analogies. Avoid jargon where possible; define it when unavoidable. British English in prose.

    **guide-what-we-measure.md** — "What We Measure and Why It Matters"
    1. **Why AI Energy Measurement Matters** — AI models consume significant energy. Understanding energy cost per inference helps inform policy, procurement, and sustainability decisions. Brief context: data centre energy use is growing, AI is a significant contributor.
    2. **Energy (Joules)** — Plain language: how much electricity a single AI inference uses. Analogy: like measuring how much fuel a car uses for a specific journey. Explain: we measure GPU power draw over time. Result is in Joules (1 Joule = 1 Watt-second).
    3. **Throughput (Tokens per Second)** — Plain language: how fast the AI produces output. Analogy: words per minute. Higher throughput = faster response. Trade-off: faster isn't always more energy efficient.
    4. **FLOPs (Floating Point Operations)** — Plain language: how much computational work the model does. Analogy: number of calculations. Larger models do more calculations. FLOPs help compare models of different sizes fairly.
    5. **Why These Three Together** — Energy alone doesn't tell the full story. A model that uses more energy but produces output 10x faster may be more efficient per unit of useful work. The three metrics together enable meaningful comparison.

    **guide-interpreting-results.md** — "How to Read llenergymeasure Output"
    1. **Result Overview** — When llenergymeasure runs, it produces a JSON result file and terminal output. This guide explains what each number means.
    2. **Key Metrics** — For each metric in the result, explain in plain language:
       - `energy_joules`: Total energy used by the GPU during inference
       - `inference_energy_joules`: Energy used for inference only (minus idle baseline)
       - `throughput_tokens_per_second`: How many tokens the model produced per second
       - `latency_seconds`: How long the inference took
       - `inference_memory_mb`: Peak GPU memory used during inference
       - `total_prompts`: How many prompts were processed
       - `total_output_tokens`: Total tokens generated
    3. **Comparing Results** — How to compare two models or configurations meaningfully. Energy per token is more useful than raw energy. Throughput matters for latency-sensitive applications. Always compare with same number of prompts and similar input lengths.
    4. **What the Numbers Mean in Practice** — Order of magnitude context: a single GPT-2 inference uses ~X Joules (very small), a large model like LLaMA-70B uses ~Y Joules. Put in context of household energy use, smartphone charging, etc. (use rough estimates, clearly labelled as approximate).

    **guide-getting-started.md** — "Running Your First Measurement (Policy Maker Guide)"
    1. **What You Need** — A computer with: Python 3.10+, an NVIDIA GPU, basic terminal/command line access. Link to installation.md for detailed setup.
    2. **Install** — `pip install "llenergymeasure[pytorch]"` — one command. Explain: this installs the measurement tool with the PyTorch inference engine.
    3. **Check Your Setup** — `llem config` — explain what the output means in simple terms.
    4. **Run Your First Measurement** — `llem run --model gpt2 --backend pytorch` — explain: this runs the GPT-2 model (a small, publicly available AI model) and measures its energy consumption. It will take a few minutes.
    5. **Read Your Results** — Walk through the terminal output line by line, explaining each metric in non-technical terms. Point to guide-interpreting-results.md for deeper explanation.
    6. **Next Steps** — Run with different models (link to researcher docs for advanced usage). Compare results across models.

    **guide-comparison-context.md** — "Comparison with Other Benchmarks"
    1. **The AI Energy Measurement Landscape** — Brief overview: several tools and benchmarks exist for AI energy/performance measurement. llenergymeasure focuses specifically on inference energy efficiency across deployment configurations.
    2. **MLPerf** — Industry standard for ML performance benchmarking. Focuses on training and inference throughput. Primarily used by hardware vendors. llenergymeasure complements MLPerf by focusing on energy, not just speed.
    3. **AI Energy Score** — Emerging benchmark for AI energy efficiency. llenergymeasure includes the AI Energy Score dataset as a built-in option. Results from llenergymeasure can contribute to or be compared with AI Energy Score benchmarks.
    4. **CodeCarbon / Zeus** — These are energy measurement libraries, not benchmarks. llenergymeasure can use both as measurement backends. CodeCarbon estimates total system energy; Zeus provides kernel-level GPU measurement. llenergymeasure orchestrates the inference workload and uses these tools for energy data.
    5. **How to Use llenergymeasure Results** — In reports: cite the tool, report configuration (model, backend, hardware, precision), report energy per token or energy per inference. In comparisons: ensure same dataset, same number of prompts, same hardware.

    These four files should be self-contained — a policy maker reading them in order should not need to read the researcher docs (though links are provided for those who want depth).
