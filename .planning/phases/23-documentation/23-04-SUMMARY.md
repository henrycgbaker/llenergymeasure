# Phase 23 Plan 04: Study Config and Technical Docs - Summary

**One-liner:** Four researcher-depth docs covering study/sweep YAML reference (7 grammar patterns), energy measurement backends, measurement methodology (warmup/baseline/thermal), and troubleshooting guide — with auto-generated content inlined.

## What Was Built

### Task 1: study-config.md and troubleshooting.md

**study-config.md** — Comprehensive YAML reference covering:
- Single experiment YAML (minimal and full examples for PyTorch and vLLM)
- All 7 sweep grammar patterns with complete copy-paste YAMLs:
  1. Single dimension sweep (`precision: [fp16, bf16]`)
  2. Multi-dimension Cartesian product (`precision` x `n`)
  3. Backend-scoped 2-segment path (`pytorch.batch_size`)
  4. Backend-scoped 3-segment path (`vllm.engine.block_size`)
  5. Explicit experiments list
  6. Base inheritance (`base: base-experiment.yaml`)
  7. Mixed sweep + explicit
- Execution configuration with concrete sequential/interleaved/shuffled examples
- Full inlined config reference between `<!-- BEGIN CONFIG REFERENCE -->` markers
- User config file schema (`~/.config/llenergymeasure/config.yaml`)

**troubleshooting.md** — Common issues in symptom/cause/fix format:
- No GPU detected, backend not installed, Docker pre-flight failures
- OOM, Docker permission denied, partial study failures
- Energy is 0 / results look wrong, warmup timing issues
- Inlined invalid combos table between `<!-- BEGIN INVALID COMBOS -->` markers
- Backend capability matrix and getting-help section

### Task 2: energy-measurement.md and methodology.md

**energy-measurement.md** — Technical depth for researchers:
- Three metric categories: energy (Joules), throughput, FLOPs with measurement semantics
- All 4 energy backends: nvml, zeus, codecarbon, null — with install instructions
- NVML measurement mechanics: 100ms polling, trapezoidal integration, timeseries storage
- Baseline power: why it matters, how it is measured and cached
- Limitations: GPU-only scope, polling frequency, multi-GPU, MIG

**methodology.md** — Statistical approach for publication-quality measurements:
- Warmup: fixed iteration and CV-based convergence detection
- Default values justified against MLPerf Power (60s thermal floor) and peer tools
- Baseline: adjusted energy formula and session caching semantics
- Multi-cycle execution: n_cycles, cycle ordering with concrete A/B/3-cycle examples
- Thermal management: gaps between experiments, throttle event recording
- Reproducibility checklist: random_seed, effective_config, model revision pinning
